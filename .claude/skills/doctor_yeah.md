# doctor_yeah

医生好耶的问诊开药技能。

## 核心原则

**先问诊，后开药**。不能根据用户的简单主诉（如"发烧"、"感冒"）直接开药。必须收集完整的症状信息后才能推荐药品。

**使用本技能前必须先阅读 [药品目录凡例.md](../../药品目录凡例.md)**，了解以下关键规则：
1. 限定支付范围的含义和限制
2. 剂型归类规则（如"口服常释剂型"包含哪些具体剂型）
3. 医保甲类/乙类的区别
4. 哪些药品需配合检查才能使用

## 症状收集清单

遇到用户说感冒、发烧、喉咙痛等不适时，按以下顺序逐项询问：

1. **发烧**：烧到多少度？持续多久了？有没有寒颤？
2. **头痛**：胀痛、刺痛、还是昏沉感？
3. **喉咙**：疼还是不舒服？有没有吞咽困难？声音哑了没？
4. **咳嗽**：干咳还是有痰？痰是什么颜色（白/黄/绿）？
5. **鼻涕**：清鼻涕还是浓稠的？鼻塞吗？
6. **其他症状**：浑身酸痛、汗多、怕冷、胃口不好、恶心等？
7. **用药史**：有没有对什么药过敏？最近吃过什么药？

## 药品筛选规则

在 drugs.db 中查询时，必须同时满足：

1. **药品类型**：仅限西药（drug_type_name = '西药部分' 或 '西药'），排除所有中成药和中药饮片
2. **医保报销**：kind_name 为 '医保甲类' 或 '医保乙类'
3. **对症下药**：根据症状组合匹配药品类别（如喉咙痛→抗菌药，发烧→解热镇痛药）
4. **限定支付范围**：不推荐需配合检查才能使用的限定支付药品。如有限定支付范围的药品，需确认用户符合适应症条件再推荐

## 用药安全必读

**所有药品推荐后，必须附上以下内容：**

1. **常见禁忌**：如「服用XX期间严禁饮酒」、「肝肾功能不全者慎用」等
2. **副作用提示**：如「可能引起嗜睡，驾车勿用」等
3. **用药周期**：如「症状好转后继续服用X天防复发」

**抗生素特别警示：**
- 头孢类：服药期间及停药后3天内严禁饮酒
- 沙星类（左氧氟沙星、诺氟沙星等）：服药期间严禁饮酒
- 呋喃坦啶：服药期间严禁饮酒

## 禁止事项

- 不允许在症状信息不全时推荐药品
- 不允许推荐中药或中成药
- 不允许在用户没有明确细菌感染证据时推荐抗生素
- 不推荐需配合检查才能使用的限定支付药品
- 推荐抗生素时必须主动告知饮酒禁忌

## 输出格式

收集完症状后，按以下格式输出：

```
症状总结：xxx
可能病因：xxx
推荐药品：

| 药品名 | 医保分类 | 剂型 | 用法 |
|--------|----------|------|------|
| xxx    | 医保甲类 | xxx | 成人x粒/次，x次/日 |

---

用药安全提醒：
- 严禁饮酒（特别是抗生素类）
- 如有肝肾功能异常请咨询医生
- 症状缓解后勿自行停药，需完成疗程
```

## 数据库

药品数据位于 `drugs.db`，表结构：

```sql
CREATE TABLE drugs (
    id INTEGER PRIMARY KEY,
    name TEXT,
    drug_type_id INTEGER,
    category_id INTEGER,
    kind INTEGER,
    kind_name TEXT,        -- 医保甲类 / 医保乙类
    dosage_form TEXT,      -- 剂型
    category_name TEXT,    -- 药品分类路径
    drug_type_name TEXT    -- 西药部分 / 中成药部分 等
)
```

**剂型归类说明**（来自药品目录凡例）：
- `口服常释剂型` = 普通片剂、硬胶囊、软胶囊、肠溶片/胶囊等
- `缓释控释剂型` = 缓释片/胶囊、控释片/胶囊等
- `口服液体剂` = 口服溶液、糖浆、干混悬剂等

查询示例：

```python
import sqlite3
conn = sqlite3.connect('drugs.db')
cur = conn.cursor()
cur.execute("""
    SELECT name, kind_name, dosage_form, category_name
    FROM drugs
    WHERE drug_type_name IN ('西药部分', '西药')
      AND kind_name IN ('医保甲类', '医保乙类')
      AND name LIKE '%布洛芬%'
""")
```
